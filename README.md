# SkillSwap – Student Skill Exchange Platform

## Project Overview
SkillSwap is a platform designed for university students to connect and exchange their skills in a collaborative peer-to-peer environment. For example, one student who excels at Python programming can team up with another student who is proficient in Graphic Design, allowing both to learn from each other's expertise. The project aims to foster a community of shared technical and creative knowledge, helping students expand their practical capabilities outside the formal classroom setting. It also serves as a practical implementation of Agile software engineering principles.

## Objectives
- Enable peer-to-peer skill exchange among students.
- Help students learn new, practical skills from their peers.
- Build a collaborative and supportive learning environment.
- Demonstrate modern Agile software development practices and methodologies.

## System Architecture
- **Frontend**: Handles the user interface and user interaction.
- **Backend**: Handles API requests and business logic.
- **Database**: MongoDB stores user data, skills, messages, and matches.
- **Authentication**: JWT manages secure login and signup processes.

## Key Features
- User Registration and Login
- Profile Creation
- Add Skills to Teach
- Add Skills to Learn
- Smart Skill Matching
- Match Requests
- Chat Between Matched Students
- Rating and Feedback System

## User Flow
1. Student creates an account.
2. Student adds skills they can teach.
3. Student adds skills they want to learn.
4. The system recommends matching students.
5. Student sends a match request.
6. The other student accepts or rejects the request.
7. Both students can communicate via chat.
8. After the session, they can rate each other.

## Agile Development Process
- **Sprint 1**: Authentication and profile system.
- **Sprint 2**: Skill matching and request system.
- **Sprint 3**: Chat and interaction features.
- **Sprint 4**: Ratings, UI improvements, and testing.

## Demo Data
Demo users were automatically generated and added to the database to simulate a fully active and realistic platform for the presentation, demonstrating how the community features function at scale.
